2.	Завдання
Підготувати доповідь на тему «Патерн проєктування Builder».
3.	Опис виконаної роботи
У процесі вивчення патерну «Будівельник» було опрацьовано його теоретичну базу за книгою «Банди чотирьох». Розглянуто структуру патерну, принцип побудови об’єктів поетапно, а також ключові компоненти: Product, Builder, ConcreteBuilder та Director.
Вивчено переваги — зручність створення складних об’єктів і гнучкість конфігурацій — та недоліки, зокрема складність у простих випадках.
Ознайомлено з прикладами застосування Builder у C#, Java (Lombok), а також у тестуванні. Реалізовано приклад на C# для практичного засвоєння матеріалу.
4.	Висновки
У ході практичної роботи було досліджено патерн Builder як ефективний засіб створення складних об’єктів із багатьма параметрами. Патерн забезпечує поетапну побудову, що підвищує гнучкість і читабельність коду.
Реалізація прикладу дозволила на практиці закріпити знання та переконатися в доцільності використання Builder у реальних задачах, особливо там, де об'єкти мають змінну структуру або багато конфігурацій.

14  public class SportsCarBuilder : ICarBuilder {
15      private Car car = new Car();
16      public void SetEngine(string engine) => car.Engine = engine;
17      public void SetWheels(int number) => car.Wheels = number;
18      public void SetGPS(bool hasGPS) => car.GPS = hasGPS;
19      public Car GetResult() => car;
20  }

22  public class Director {
23      public void BuildSportsCar(ICarBuilder builder) {
24          builder.SetEngine("V8");
25          builder.SetWheels(4);
26          builder.SetGPS(true);
27      }
28  }
